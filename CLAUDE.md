## Главные правила

### Server-First (данные на сервере)
- Страницы по умолчанию — серверные (Server Components)
- `'use client'` — только где нужна интерактивность (клики, ввод, анимации)
- Данные загружаются на сервере, НЕ через useEffect
- Это даёт SEO, быструю загрузку, безопасность

### НЕТ inline стилям
- `style={{ color: 'red' }}` — **ЗАПРЕЩЕНО**
- Всё через Tailwind CSS классы: `className="text-red-500"`

### Типы из базы данных
- Типы генерируются автоматически: `pnpm db:types`
- НЕ писать типы вручную для данных из БД

Все правила → `docs/RULES.md`

## Git — СТРОГИЕ правила

### Коммиты
- **Делай коммит** после каждого завершённого блока работы
- Используй понятные сообщения на английском: `feat: add login page`, `fix: broken navigation`
- Префиксы: `feat:` (новое), `fix:` (исправление), `refactor:` (улучшение), `chore:` (настройки), `docs:` (документация)

### Пуш (загрузка на GitHub)
- **НИКОГДА не делай push без команды пользователя**
- После коммита ВСЕГДА напоминай: *"Изменения сохранены локально. Хочешь загрузить на GitHub? (после загрузки проект автоматически обновится на сервере)"*
- Жди явного подтверждения перед push

### Безопасность
- **НИКОГДА** не коммить файлы с паролями (`.env.local`, `ACCESS.md`)
- Проверяй `.gitignore` перед каждым коммитом
