## Проблема 1: Файл-монстр — admin/page.tsx (3 797 строк)

**Путь**: `platform/src/app/(platform)/admin/page.tsx`

Один файл содержит ВСЁ:
- 77 вызовов useState (77 переменных состояния)
- 517 inline стилей вроде `style={{ position: 'fixed', inset: 0, zIndex: 99999 }}`
- Захардкоженные данные курсов (все модули и уроки — 400 строк)
- Дублирование типов (уже есть в types.ts, но написаны заново)
- Модальные окна, SVG иконки, CRUD логика — всё в одном файле
- Рендер всех 15 вкладок в одном switch-case

**В новом проекте**: каждая вкладка — отдельная страница с URL. Никаких файлов больше 300 строк.

## Проблема 2: Всё на клиенте — нет серверного рендеринга

Каждая страница начинается с `'use client'` и грузит данные через `useEffect`:

```tsx
// ТАК ДЕЛАТЬ НЕЛЬЗЯ — каждая страница старого проекта написана так:
'use client'
export default function Page() {
  const [data, setData] = useState(null)
  useEffect(() => { loadData().then(setData) }, [])
  return data ? <Content /> : <div>Загрузка...</div>
}
```

**Последствия**:
- Google не видит содержимое страниц (нулевой SEO)
- Пользователь видит пустой экран пока грузятся данные
- Нет кеширования

**В новом проекте**: данные грузятся на сервере (Server Components). Пользователь сразу видит готовую страницу.

## Проблема 3: CSS хаос

- **5 413 строк** в 9 CSS файлах без изоляции — все классы глобальные
- **112 `!important`** в layout.tsx для мобильных стилей (в теге `<style>` внутри `<head>`)
- Inline стили дублируют CSS (стили написаны дважды — и в CSS, и в style={})
- Имена классов конфликтуют: `.card`, `.main`, `.aside`, `.val`, `.lbl`

**В новом проекте**: Tailwind CSS — стили через классы, без конфликтов, без `!important`.

## Проблема 4: Фейковая безопасность

**Путь**: `platform/src/lib/security.ts`

Что там написано:
- "Запрет Ctrl+U, Ctrl+S, PrintScreen" — обходится через DevTools за 1 секунду
- "Rate limiter" — хранит счётчик в переменной браузера (обнуляется при перезагрузке)
- "Проверка реферера" — проверяет адрес в браузере (легко подменить)
- "Водяной знак" — текст на клиенте (виден в DOM, легко удалить)

**В новом проекте**: защита на сервере — signed URLs для видео, серверный rate limiting, серверная авторизация.

## Проблема 5: Нет инфраструктуры

Чего нет:
- Тестов (ноль)
- CI/CD (нет автодеплоя — файлы загружались на сервер вручную)

## Проблема 6: Всё в одном приложении

Лендинг + админка + платформа студента — в одном Next.js приложении:
- Код админки увеличивает размер лендинга
- CSS файлы конфликтуют между собой
- Невозможно деплоить отдельно

**В новом проекте**: 3 отдельных приложения в монорепо.

## Проблема 7: Данные без типов

**Путь**: `platform/src/lib/api.ts`

- Функции обновления принимают `Record<string, unknown>` — можно передать что угодно
- Типы написаны вручную и могут расходиться с реальной базой данных
- Нет серверного Supabase клиента — всё через anon key в браузере

**В новом проекте**: типы генерируются автоматически из базы данных.
